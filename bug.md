# Bug 修复记录

> 记录项目从原始 JS 版本迁移至 Vue 3 过程中发现并修复的全部问题

## 样式问题

1. ✅ **添加聊天时间：输入框时分相隔很远**
   - **修复**：添加了 `picker-seg` flex 布局和 `.chat-time-picker` 紧凑排列样式
   - **状态**：✓ 已修复

2. ✅ **布局有问题，输入配置和微信聊天界面应该是左右布局**
   - **修复**：添加了 `.page-content` flex 布局（`display: flex; flex-direction: row`）
   - **状态**：✓ 已修复

## 功能问题

1. ✅ **外观设置无法点击**
   - **修复**：添加了 Vue 的 `activeTab` 状态 + zui 原生的 `.active` 类机制控制 Tab 切换
   - **状态**：✓ 已修复

2. ✅ **两个用户都被勾选，发送人显示第一个用户**
   - **修复**：第二个用户初始化时已设为 `selected: false`
   - **状态**：✓ 已修复
   - **附注**：用户选择徽标样式已统一为 CSS 对勾伪元素（`.user-check-badge.checked::after`）

3. ✅ **勾选用户未生效，发送人固定为第一个用户**
   - **修复**：所有 `add*Dialog` 函数使用 `getSelectedUser()` 动态获取发送者并正确设置 `is_me`
   - **状态**：✓ 已修复

4. ✅ **无法添加时间，弹窗提示"请输入时间内容"**
   - **修复**：`addNoticeDialog` 重构为构建时间字符串，支持通过日期/时间选择器联动
   - **状态**：✓ 已修复

5. ✅ **红包或转账金额输入框没有默认值**
   - **修复**：默认值改为 `88`
   - **状态**：✓ 已修复

6. ✅ **红包/转账领取后缺少对方记录**
   - **修复**：
     - `redpacketGet`：在当前记录标记已领取，同时在对方聊天记录追加一条已领取的红包卡片
     - `transferGet`：当前记录变为"已被接收"，同时在对方聊天记录追加一条"已收款"的转账卡片
   - **状态**：✓ 已修复

7. ✅ **转账默认备注"请收款"未展示**
   - **修复**：默认值改为 `"请收款"`
   - **状态**：✓ 已修复

## 额外修复

8. ✅ **Tab 切换不显示（zui.min.css 与 Vue v-show 冲突）**
   - **根因**：`zui.min.css` 中 `.tab-content>.tab-pane { display: none }` 与 Vue `v-show` 的 inline `display` 冲突
   - **修复**：改用 zui 原生的 `.active` 类（`:class="{ active: condition }"`）控制 tab 显示
   - **状态**：✓ 已修复
